# 项目新对话摘要

## 写作约束

本项目虽然是在既有后台管理框架基础上改造完成的，但在论文撰写、项目介绍和系统设计说明中，不要提及 RuoYi 或若依。论文中应统一表述为“基于 Spring Boot 与 Vue 的前后端分离医疗检测系统”或“肠道息肉智能检测系统”。如果需要说明基础能力，可以描述为系统实现了用户认证、权限控制、菜单管理、日志审计和文件资源管理等通用后台能力，不追溯具体开源框架来源。

项目介绍应围绕当前实际运行的息肉检测主线展开。项目中保留了一些旧 HIS 或通用后台代码，但未纳入当前系统核心业务流程的功能、页面和数据表，不应在论文或新对话中作为已实现功能展开描述。

## 项目定位

本项目是一个面向内镜检查场景的肠道息肉智能检测系统，主要用于辅助医护人员对内镜图片或视频进行息肉识别、结果查看和检测任务管理。系统通过上传检查影像资料、调用 YOLOv5 推理服务、保存检测结果和展示识别明细，形成从“资料上传”到“检测结果查看”的完整业务闭环。

系统的主要使用者是医护人员和系统管理员。医护人员关注检测任务提交、识别结果查看、历史任务查询和检测数据统计；系统管理员关注账号、组织、权限和日志审计等基础维护工作。

## 技术架构

系统采用前后端分离架构。前端使用 Vue 和 Element UI 构建页面，负责登录、监测总览、息肉检测中心、任务列表、结果展示和基础管理界面。后端使用 Spring Boot、Spring Security、JWT、MyBatis 和 MySQL，负责身份认证、权限校验、文件上传、检测任务创建、任务状态维护、结果查询、数据导出和日志记录。

息肉识别能力由独立的 Python FastAPI 推理服务提供。后端在创建检测任务后，通过 HTTP 接口调用推理服务，将上传的图片或视频发送给 YOLOv5 模型进行识别。推理服务返回检测框坐标、置信度、结果图片、结果视频、命中帧 JSON 等信息，后端再将结果写入数据库，供前端展示和统计。

## 实际使用功能

系统实际使用的前台业务功能主要包括监测总览和息肉检测中心。

监测总览用于展示系统检测任务运行情况，包括总检查任务数、今日检查任务数、成功任务数、失败任务数、平均推理耗时、累计识别息肉数量、最近 7 天任务趋势、图片与视频任务占比、任务状态分布和最近任务列表。该页面帮助用户快速了解系统当前检测工作量和识别结果概况。

息肉检测中心是系统核心业务页面。用户可以上传图片或视频文件，系统校验文件格式和大小后保存文件资源，并创建检测任务。任务创建后进入后台异步执行流程，前端定时轮询任务状态。任务成功后，系统展示检测结果图或结果视频，并列出息肉检测框明细，包括类别、置信度和坐标信息。用户还可以按任务编号、状态、媒体类型和创建时间筛选历史任务，对任务进行查看、删除和导出。

系统基础管理功能包括登录认证、用户权限控制、医护账号管理、组织管理、登录审计和操作审计。登录认证用于保证系统访问安全；权限控制用于限制不同角色可访问的菜单和接口；审计日志用于记录登录、上传、创建任务、查看详情、删除和导出等关键行为。

## 核心业务流程

用户登录系统后进入监测总览页面，查看检测任务统计数据和最近任务。需要进行检测时，用户进入息肉检测中心，选择待检测的内镜图片或视频并提交检查。系统首先将文件保存为文件资源记录，然后根据当前启用的检测模型创建检测任务。任务进入待处理或处理中状态后，后端异步调用 Python 推理服务。

推理服务接收文件后调用 YOLOv5 模型完成识别。对于图片，系统生成带检测框的结果图片，并返回息肉位置和置信度。对于视频，系统按帧进行检测，生成结果视频和命中帧数据。后端接收推理结果后，更新检测任务状态，写入检测结果汇总和息肉详情。前端轮询到任务成功后，展示结果媒体和检测明细。

如果推理服务不可用、文件不存在或识别过程异常，任务会被标记为失败，并保存失败原因，便于用户查看和管理员排查。

## E-R 图对应的实际数据实体

结合系统实际使用情况和 E-R 图，当前核心实体包括用户、部门、息肉检测模型、文件资源、息肉检测任务、检测结果、息肉详情、登录日志和操作日志。

用户归属于部门，用户可以创建多个息肉检测任务。每个检测任务使用一个源文件和一个检测模型。检测任务执行成功后生成一条检测结果，一条检测结果可以包含多条息肉详情。用户登录行为写入登录日志，文件上传、任务创建、查看详情、删除和导出等业务行为写入操作日志。

实际使用的数据表主要包括：

- `sys_user`：用户/医护账号表，保存账号、昵称、密码、部门、状态等信息。
- `sys_dept`：部门/组织表，保存医院内镜中心、科室和组织层级信息。
- `sys_role`、`sys_user_role`、`sys_menu`、`sys_role_menu`：权限支撑表，用于角色、菜单和接口权限控制。
- `sys_logininfor`：登录审计表，记录登录成功、失败和退出等访问行为。
- `sys_oper_log`：操作审计表，记录上传、创建任务、查看详情、删除、导出等关键操作。
- `file_asset`：文件资源表，保存上传图片或视频的原始文件名、扩展名、大小、存储路径和访问 URL。
- `polyp_model`：息肉检测模型表，保存模型名称、版本、模型路径、启用状态、默认模型标记和置信度阈值。
- `polyp_detect_task`：息肉检测任务表，保存任务编号、用户、源文件、模型、状态、媒体类型、推理耗时、结果 URL、帧统计、检测框数量和最大置信度。
- `his_detection_result`：检测结果汇总表，保存检测类型、样本编号、原始文件、结果文件、息肉数量、尺寸统计、平均置信度和检测时间。
- `his_polyp_detail`：息肉详情表，保存每个息肉检测框的序号、置信度、尺寸、坐标、宽高和分类。

论文中可以将本项目概括为：本系统设计并实现了一套基于深度学习的肠道息肉智能检测平台，采用前后端分离架构，将医疗影像资料管理、检测任务管理、YOLOv5 模型推理、检测结果展示和审计管理整合到同一系统中。系统支持内镜图片和视频上传检测，能够展示检测结果图、结果视频、息肉目标框、置信度和历史任务统计，为医护人员提供辅助筛查和结果追溯能力。
